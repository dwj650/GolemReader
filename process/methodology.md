# Armature in brief

The human is the **client**; the AI is the **firm**. The full process runs behind the
curtain; the operator sees plain-language recommendations to approve.

Work hierarchy: **Foundation (G0) → Version (G5) → Phase (G1/G4) → Step (G2/G3)**, and a
step climbs the **Rungs**. Three **zones**: Design (chat), Build (coding agent),
Validation (terminal/device/CI). Three **actors**: Human, Chat Assistant, Coding Agent.
Work crosses zones only via a **Baton**.

Operative details live in the sibling files: `rungs.md`, `gates.md`, `guards.md`,
`collaboration.md`, `response-contract.md`, `coding-standards.md`, `anti-patterns.md`,
`glossary.md`. The single source of truth for *current state* is
`state/current-state.md`; the ID registry is `index.md`.

The full reference (rationale + the locked charter D1–D32) is the canonical Armature
reference document. This repo's `process/` files are the working checklists derived
from it.
