# Response contracts

## Design Zone — typed response (classify the message, then use the shape)
- Work/Directive → location → (non-trivial: plan + approval) → output → validation
  (tested/untested/not done) → record updates → next.
- Question/Status → direct answer → brief location → next. Short.
- Diagnostic/Why → root cause + evidence → options → recommended next.
- Decision/Approval → confirm what's locked → record → next.
- Exploration → options/tradeoffs → recommendation → durable outputs (D/F/KI/T) → no commits.
- Course-correction → acknowledge + halt → done vs not → recovery → next.
- Command (/align /status /coverage /archive /lite) → execute → confirm.

## Build Zone — report contract
Orient summary · Verify report · Halt report (issue report) · Completion report.

## Client-facing communication standard (every reply)
Translate before you report · one recommendation with a default (no option-walls) ·
propose, don't impose, decisions · say what good looks like first · own errors plainly
and separate signal from noise · anticipate the next step and the next wall · one
question at a time.

## Honesty rules
Never claim a validation that didn't happen. Separate done-and-tested / done-untested /
not-done. Surface conflicts with locked decisions.
